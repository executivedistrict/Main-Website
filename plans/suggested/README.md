# Suggested plans

Plans promoted from `../deferred/` because a newly-drafted plan identified
them as a recommended predecessor or immediate follow-on.

**Why plans land here:**
- During the kickoff of a new plan, the orchestrator reviewed `../deferred/`
  and flagged a plan that should either ship BEFORE the new one (unblocking
  it) or RIGHT AFTER (because the new plan leaves obvious gaps the deferred
  plan closes).
- Any promotion to `suggested/` is cross-referenced in the current plan's
  `README.md` under a "Related plans" section so reviewers see the link.

**Lifecycle:**
- `deferred/` → `suggested/` when flagged as a near-term dependency
- `suggested/` → top-level (`plans/NNN-...`) when scheduled to execute
- `suggested/` → `deferred/` if priority drops again

See `../AGENTS.md` for the full review procedure.
